# 커밋 메시지 가이드

## 형식

```
<type>: <제목>

<본문> (선택)
```

- 제목은 50자 이내, 마침표 없이 작성
- 본문은 **무엇을**이 아닌 **왜**를 설명

---

## Type 목록

| Type | 설명 |
|------|------|
| `feat` | 새 기능 추가 |
| `fix` | 버그 수정 |
| `refactor` | 동작 변경 없는 코드 구조 개선 |
| `test` | 테스트 추가/수정 |
| `docs` | 문서 작성/수정 |
| `chore` | 빌드 설정, 패키지 등 기타 변경 |

---

## 예시

```
feat: 주문 금액 합산 기능 추가

할인율 적용 전 금액을 기준으로 합산하도록 변경.
기존 방식은 할인 후 금액이 누적되어 총액이 맞지 않는 문제가 있었음.
```

```
fix: 수량 0 입력 시 크래시 수정
```

```
refactor: Order 클래스에서 가격 계산 로직 분리
```

---

## 주의

- 구현 중인 작업은 커밋하지 않는다 — Phase 단위로 완료 후 커밋
- 하나의 커밋에 여러 관심사를 섞지 않는다
- `fix: 수정함`, `feat: 기능` 같은 의미 없는 제목은 작성하지 않는다
