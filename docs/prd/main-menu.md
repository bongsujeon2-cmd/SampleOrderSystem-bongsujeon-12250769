# PRD: 메인 메뉴 (Main Menu)

> **관련 문서**
> - PLAN: [docs/plan/main-menu.md](../../plan/main-menu.md)
> - Design: [docs/design/main-menu.md](../../design/main-menu.md)

---

## 배경 및 목적

담당자가 시스템 진입 시 전체 시료 재고 현황을 즉시 파악하고, 원하는 기능으로 빠르게 이동할 수 있는 중앙 허브 화면을 제공한다. 메뉴 루프를 순환할 때마다 생산 완료 여부를 자동으로 확인하여 담당자가 별도 조작 없이도 주문 상태가 최신으로 유지되도록 한다.

---

## 사용자 스토리

- 담당자로서, 메인 화면에서 전체 시료의 재고 현황 요약을 즉시 확인하고 싶다.
- 담당자로서, 번호를 입력하여 원하는 기능으로 빠르게 이동하고 싶다.
- 담당자로서, 메뉴를 선택할 때마다 생산 완료된 주문이 자동으로 처리되어 있길 원한다.

---

## 기능 요구사항

### 재고 현황 스냅샷 표시

- 메인 화면 진입 시 등록된 전체 시료의 재고 현황 요약 정보를 표시한다.

### 생산 완료 Lazy 자동 체크

- 메인 루프를 한 번 순환할 때마다 현재 activeJob의 완료 여부를 자동으로 판단한다.
- 완료 조건: `현재 시각 - startTimeUnix(초) >= 총 생산 시간(분) × 60`
- 완료된 작업이 있으면 자동 처리 후 다음 대기 작업을 시작한다 (상세 처리 순서는 생산 라인 PRD 참조).
- 이 체크는 담당자의 별도 메뉴 진입 없이도 실행된다.

### 메뉴 선택

| 번호 | 메뉴 | 설명 |
|------|------|------|
| 1 | 시료 관리 | 시료 등록 / 목록 조회 / 검색 |
| 2 | 주문 | 주문 접수 / 승인·거절 처리 |
| 3 | 모니터링 | 상태별 주문 수 및 재고 현황 |
| 4 | 출고 처리 | CONFIRMED 주문 출고 실행 |
| 5 | 생산 라인 | 현재 생산 현황 및 대기 큐 확인 |
| 0 | 종료 | 프로그램 종료 |

---

## 비기능 요구사항

- 메뉴 루프 반복마다 생산 완료 체크를 수행하여 주문 상태가 항상 최신 상태로 유지되어야 한다.
- 화면 구성은 자유롭게 설계하되, 재고 현황 스냅샷과 메뉴 항목이 한 화면에 표시되어야 한다.

---

## 성공 기준

- 메인 화면에서 전체 시료의 재고 현황이 표시된다.
- 메뉴 루프를 순환할 때 생산이 완료된 작업이 있으면 자동으로 CONFIRMED 전환 처리된다.
- 0~5번 이외의 입력에는 오류 메시지를 출력하고 메뉴를 다시 표시한다.
- 0번 입력 시 프로그램이 정상 종료된다.
