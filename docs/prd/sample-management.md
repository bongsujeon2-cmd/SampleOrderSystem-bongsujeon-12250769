> - Design: [docs/design/sample-management.md](../../design/sample-management.md)

---

## 배경 및 목적

시스템의 기본 단위인 시료(Sample)를 등록·조회·검색한다. 등록된 시료만 주문 가능하며, 시료별 재고 현황을 관리한다.

---

## 사용자 스토리

- 담당자로서, 새 시료를 등록하여 주문 가능한 품목 목록에 추가하고 싶다.
- 담당자로서, 등록된 모든 시료와 현재 재고 수량을 한눈에 확인하고 싶다.
- 담당자로서, 시료 이름으로 검색하여 원하는 시료 정보를 빠르게 찾고 싶다.

---

## 기능 요구사항

### 시료 등록

| 입력 필드 | 타입 | 설명 |
|-----------|------|------|
| 시료 ID | string | 사용자가 직접 입력하는 고유 식별자 |
| 시료 이름 | string | 시료명 |
| 평균 생산시간 | int | 시료 1개 생산 소요 시간 (단위: 분) |
| 수율 | double | 정상 시료 / 총 생산 시료 (0 초과 ~ 1.0 이하) |

- 동일 ID 또는 동일 이름 중복 등록 방지
- 등록 성공 시 `data/samples.json`에 즉시 저장

### 시료 목록 조회

- 등록된 모든 시료 목록 출력
- 각 시료의 현재 재고 수량 함께 표시

### 시료 검색

- 시료 이름 부분 일치 검색
- 결과 표시: 시료 ID, 이름, 수율, 평균 생산시간, 재고 수량

---

## 비기능 요구사항

- 앱 재시작 후에도 등록된 시료 데이터가 유지되어야 한다 (JSON 영속성)
- 수율은 0 초과 1.0 이하 값만 허용한다

---

## 성공 기준

- 시료를 등록하면 즉시 주문 가능 목록에 나타난다
- 중복 ID/이름으로 등록 시 오류 메시지가 출력되고 등록이 거부된다
- 앱 재시작 후 기존 시료 데이터가 그대로 유지된다
