## 배경 및 목적

담당자가 시스템 전체 주문 상태와 시료별 재고 현황을 한눈에 파악할 수 있도록 한다.

---

## 사용자 스토리

- 담당자로서, 상태별 주문 건수와 목록을 확인하여 처리 현황을 파악하고 싶다.
- 담당자로서, 시료별 재고 상태(여유/부족/고갈)를 한눈에 확인하고 싶다.

---

## 기능 요구사항

### 주문량 확인

- 상태별 주문 목록 및 건수 표시
- 포함 상태: **RESERVED / PRODUCING / CONFIRMED / RELEASE**
- 제외 상태: REJECTED

### 재고량 확인

| 재고 상태 | 조건 | 표기 |
|-----------|------|------|
| 여유 | 현재 재고 > 유효 주문 수량 합계 | [여유] |
| 부족 | 0 < 현재 재고 ≤ 유효 주문 수량 합계 | [부족] |
| 고갈 | 현재 재고 = 0 | [고갈] |

> **유효 주문 수량 합계 정의:** 해당 시료에 대해 **RESERVED / PRODUCING / CONFIRMED** 상태인 주문의 수량 합계.
> RELEASE(출고 완료)는 이미 재고가 차감되므로 제외하고, REJECTED는 취소된 주문이므로 제외한다.

- 각 시료별 현재 재고 수량과 상태 표시

---

## 비기능 요구사항

- 조회 시점의 최신 데이터를 반영해야 한다

---

## 성공 기준

- REJECTED 주문이 집계에서 제외된다
- 시료별 재고 상태가 여유/부족/고갈 중 하나로 정확히 표시된다
