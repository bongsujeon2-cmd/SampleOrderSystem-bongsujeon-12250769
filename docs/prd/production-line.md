# PRD: 생산 라인 (Production Line)

> **관련 문서**
> - PLAN: [docs/plan/production-line.md](../../plan/production-line.md)
> - Design: [docs/design/production-line.md](../../design/production-line.md)

---

## 배경 및 목적

재고 부족으로 승인된 주문을 생산 큐에서 순차 처리한다. 생산 완료 여부를 실시간 unix timestamp 비교로 판단하며, 프로그램이 꺼져 있던 동안의 시간도 반영한다.

---

## 사용자 스토리

- 담당자로서, 현재 생산 중인 작업의 진행 상황(경과 시간, 예상 완료 시각)을 확인하고 싶다.
- 담당자로서, 생산 대기 중인 주문 목록과 순서를 확인하고 싶다.
- 담당자로서, 앱을 재시작해도 생산 진행 상태가 유지되고 완료된 작업이 자동 처리되길 원한다.

---

## 기능 요구사항

### 생산 현황 표시 (Active Job)

| 표시 항목 | 내용 |
|-----------|------|
| 주문 ID | 어떤 주문에 대한 생산인지 |
| 시료명 | 생산 중인 시료 |
| 실 생산량 | 목표 생산 수량 |
| 총 생산 시간 | 예상 소요 시간 (분) |
| 생산 시작 시각 | 생산 시작 unix timestamp |
| 경과 시간 | 현재까지 경과된 시간 (분) |
| 예상 완료 시각 | 시작 시각 + 총 생산 시간 |

### 생산 완료 처리 (Lazy)

- **체크 트리거:** 메인 루프 반복 시, 생산 라인 메뉴 진입 시
- **완료 판단:** `현재 시각 - startTimeUnix(초) >= 총 생산 시간(분) × 60`
- 완료 시 처리 순서:
  1. 실 생산량 전부를 해당 시료 재고에 추가
  2. 주문 상태: PRODUCING → CONFIRMED
  3. activeJob 제거 및 JSON 저장
  4. 큐에 대기 항목이 있으면 다음 작업 자동 시작
  5. 새로 시작된 작업도 즉시 완료 조건 재체크

### 대기 주문 확인 (Queue)

- 생산 큐 대기 목록 출력 (FIFO 순서)
- 표시: 대기 순번, 주문 ID, 시료명, 주문 수량, 실 생산량, 예상 소요 시간(분)

### 시간 Mocking (테스트 지원)

- `--mock-time` 인수로 MockTimeProvider 활성화
- MockTimeProvider의 `advance(minutes)`로 시각을 임의로 앞당겨 빠른 검증 가능
- 모든 시간 연산은 ITimeProvider 인터페이스를 통해서만 수행

---

## 비기능 요구사항

- 프로그램 재시작 후 기존 생산 진행 상태(startTimeUnix)가 유지되어야 한다
- 재시작 시 꺼져 있던 동안 완료된 생산 작업을 자동으로 일괄 처리해야 한다
- 생산 라인은 한 번에 하나의 작업만 처리한다 (단일 생산 라인)

---

## 성공 기준

- 생산 시간이 경과하면 메뉴 진입 시 자동으로 CONFIRMED 전환된다
- 앱을 재시작해도 생산 시작 시각 기반으로 완료 여부가 정확히 판단된다
- `--mock-time` 모드에서 시간을 앞당기면 생산 완료가 즉시 처리된다
- 큐에 여러 작업이 쌓인 경우 FIFO 순서대로 순차 처리된다
