---

## 배경 및 목적

개발 및 테스트 단계에서 JSON 파일에 저장된 데이터를 빠르게 확인하고, 반복적인 수동 입력 없이 테스트용 더미 데이터를 자동 생성하기 위한 별도 관리자 도구를 제공한다.

---

## 사용자 스토리

- 개발자로서, 현재 JSON 파일에 저장된 시료·주문·생산 데이터를 콘솔에서 즉시 조회하여 상태를 확인하고 싶다.
- 개발자로서, 테스트에 필요한 더미 데이터를 자동으로 생성하여 JSON 파일에 저장함으로써 반복적인 수동 입력 시간을 줄이고 싶다.

---

## 기능 요구사항

### 데이터 모니터링 Tool

- 현재 `data/samples.json`, `data/orders.json`, `data/production.json`에 저장된 모든 데이터를 콘솔에 출력한다.
- 별도 실행 또는 메인 프로그램 내 숨김 명령으로 접근할 수 있다.
- 출력 형식은 자유롭게 설계하되, 각 JSON 파일의 전체 내용이 사람이 읽기 쉬운 형태로 표시되어야 한다.

### Dummy 데이터 생성 Tool

- 미리 정의된 시나리오에 따라 테스트용 시료·주문·생산 데이터를 자동 생성한다.
- 생성된 데이터는 `data/samples.json`, `data/orders.json`, `data/production.json`에 즉시 저장된다.
- 기존 데이터를 덮어쓰거나 추가하는 방식을 선택할 수 있어야 한다.
- 생성되는 더미 데이터는 PRD에 정의된 비즈니스 규칙(BR-01~BR-18)을 준수하는 유효한 데이터여야 한다.

---

## 비기능 요구사항

- 두 도구는 메인 프로그램(`SampleOrderSystem.exe`)과는 독립적으로 실행 가능하거나, 별도 인수(예: `--dump-data`, `--gen-dummy`)로 활성화된다.
- 더미 데이터 생성 후 메인 프로그램을 실행했을 때 정상 동작해야 한다.

---

## 성공 기준

- 데이터 모니터링 Tool 실행 시 세 JSON 파일의 현재 저장 내용이 콘솔에 출력된다.
- Dummy 데이터 생성 Tool 실행 시 유효한 시료·주문·생산 데이터가 JSON 파일에 저장되고, 이후 메인 프로그램에서 정상적으로 로드된다.
