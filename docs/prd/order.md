# PRD: 주문 접수 및 승인/거절 (Order)

> **관련 문서**
> - PLAN: [docs/plan/order.md](../../plan/order.md)
> - Design: [docs/design/order.md](../../design/order.md)

---

## 배경 및 목적

고객이 시료를 주문하면 담당자가 승인/거절을 처리한다. 승인 시 재고 상황에 따라 즉시 출고 대기 상태가 되거나 생산 큐에 등록된다.

---

## 사용자 스토리

- 고객으로서, 원하는 시료를 수량과 함께 주문하여 접수 확인을 받고 싶다.
- 담당자로서, 접수된 주문 목록을 확인하고 승인 또는 거절을 처리하고 싶다.
- 담당자로서, 승인 시 재고 상황에 따라 자동으로 출고 대기 또는 생산 등록이 되길 원한다.

---

## 기능 요구사항

### 주문 접수 (예약)

| 입력 필드 | 타입 | 설명 |
|-----------|------|------|
| 시료 ID | string | 등록된 시료만 허용 |
| 고객명 | string | 주문 고객 이름 |
| 주문 수량 | int | 양의 정수 |

- 접수 완료 시 주문 상태: **RESERVED**
- 주문 ID 자동 생성 (형식: `ORD-NNN`)
- 미등록 시료 ID 입력 시 오류 메시지 출력

### 접수된 주문 목록 조회

- RESERVED 상태 주문만 표시
- 표시 항목: 주문 ID, 시료명, 고객명, 수량, 접수 일시

### 주문 승인

재고 상황에 따라 자동 분기:

```
현재 재고 >= 주문 수량
    → RESERVED → CONFIRMED (재고 차감 없음, 출고 시 차감)

현재 재고 < 주문 수량
    → 부족분 = 주문 수량 - 현재 재고
    → 실 생산량 = ceil(부족분 / (수율 × 0.9))
    → 총 생산 시간 = 평균 생산시간(분) × 실 생산량
    → 생산 큐(FIFO)에 등록
    → RESERVED → PRODUCING
```

### 주문 거절

- 선택한 주문 즉시 REJECTED 전환
- REJECTED 주문은 이후 모니터링에서 제외

---

## 비기능 요구사항

- 재고 차감은 출고(RELEASE) 시점에만 발생한다 (승인 시 차감 없음)
- 동일 시료에 대한 여러 PRODUCING 주문은 각각 독립적인 생산 작업으로 처리된다

---

## 성공 기준

- 주문 접수 후 RESERVED 상태로 목록에 표시된다
- 재고 충분 시 승인하면 즉시 CONFIRMED로 전환된다
- 재고 부족 시 승인하면 PRODUCING으로 전환되고 생산 큐에 등록된다
- 거절된 주문은 모니터링 집계에서 제외된다
